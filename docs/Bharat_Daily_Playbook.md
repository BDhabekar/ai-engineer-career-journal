## EVERY WEEKDAY (Monday–Friday) — do these steps in order

### Step 1 — 7:00 AM
1. Sit down at your study spot.
2. Open a **brand-new chat** with Claude. (Not yesterday's chat. A new one.)
3. Upload all 6 files: Tracker PDF, Tracker Excel, `roadmap.md`, `memory-log.md`, this Daily Playbook, and the **active project's `README.md`** (e.g. the `01-ml-foundations-iris` README).
4. Type exactly: `continue`
5. Claude will teach you. Just follow along — read, run the code, ask questions if something is confusing.
6. Stop at 8:45 AM, even if you're not fully done. That's okay.

### Step 2 — 8:45–9:15 AM
1. Write 2–3 sentences: "Today I learned ___."
2. Write down any questions you still have.
3. Get ready and go to work.

### Step 3 — 11:00 AM – 6:00 PM
1. Do your job. Don't think about studying right now. This time is not yours.

### Step 4 — 7:45–8:45 PM
1. Continue the same chat from this morning (or open it again).
2. Do light coding, or one LeetCode problem (check `roadmap.md` Section 7 for how many today), or re-read today's concept.

### Step 5 — 9:00–9:30 PM (Closing the day — don't skip this)
1. In the chat, type exactly: `End of day`
2. Claude will hand you back a set of updated files to save. **Every day** this includes:
   - **today's daily note** → save as a new dated file, like `2026-06-25.md`, inside the `daily/` folder of the journal repo
   - **updated `memory-log.md`** → replace the old one in the journal repo's `docs/` folder
   - **updated `roadmap.md`** → replace the old one in the journal repo's `docs/` folder
   - **updated `STATUS.md`** → overwrite it at the journal repo root
   - **updated project `README.md`** → replace it in the active project repo (e.g. `01-ml-foundations-iris`)
3. **If you solved a LeetCode/DSA problem today**, Claude also hands you:
   - the **solution file**, using the fixed header template (see `memory-log.md` Section 3 / `roadmap.md` Section 7) → save it in the right pattern folder of your `dsa-practice` repo
   - the **updated `dsa-practice` README** progress-table row → replace it in that repo
4. Save everything to GitHub (commit) with short messages, like `day2: linear regression from scratch` and `dsa: valid-anagram (arrays-hashing)`. The three repos (journal, active project, `dsa-practice`) each get their own commit.
5. Push it all to GitHub.

### Step 6 — 9:30 PM onward
1. Stop all work. Completely.
2. Sleep on time.

### Step 7 — Tomorrow
1. Go back to Step 1. New day, new chat.

## GOLDEN RULES — never break these
1. **Always** start with a brand-new chat and upload all 6 files. Every single day, no exceptions.
2. **Always** say `End of day` before you stop. If you skip this, tomorrow's chat won't know what happened today.
3. **Never** skip the evening commit/push to GitHub.
4. If you feel confused about what to do — just type `continue`. Claude will figure out where you are from the 6 files.
5. **Missed a whole day?** No problem. Next day, just open a new chat, upload the 6 files, type `continue`. Don't try to "catch up" by doing two days at once.
6. **Never edit `roadmap.md` or `memory-log.md` yourself.** Only Claude changes these, and only when you ask it to (usually during "End of day"). You just save the file it gives you.
