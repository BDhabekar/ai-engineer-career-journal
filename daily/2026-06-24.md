# 2026-06-24 — Day 2: Linear & Logistic Regression

**Objective:** Understand linear/logistic regression math + gradient descent; implement linear regression from scratch in NumPy; connect regularization and evaluation metrics back to Day 1's Iris baseline.

**What was actually done:**
- Built `LinearRegressionScratch` (batch gradient descent) in NumPy on a synthetic dataset; verified against the closed-form normal equation and `sklearn.LinearRegression` — all three agreed to 4 decimal places (θ₀≈4.0842, θ₁≈2.9688).
- Compared batch vs. mini-batch vs. SGD convergence noise on the same data and learning rate.
- Demonstrated L1 vs. L2 on a 10-feature synthetic set with 6 known-useless features — Lasso zeroed exactly those 6; Ridge only shrank them.
- Reapplied Day 1's exact Iris split/model to compute precision/recall/F1/ROC-AUC (macro AUC = 1.0), not just accuracy.
- Notebook (`02_linear_regression_scratch.ipynb`) + 4 plots built, executed clean top-to-bottom, placed for `01-ml-foundations-iris/notebooks/` and `/images/` — ready to commit.
- DSA: Valid Anagram (LeetCode 242, arrays-hashing). Solved independently with a two-pass hashmap approach (correct, O(n) time). Explored two more angles for depth: a fixed-array O(1)-space version (exploits the lowercase-only constraint), and a one-pass hashmap version (zip both strings into one loop). Committed file contains the hashmap + array versions, as requested; the one-pass variant was discussed in chat only and is not in the file.

**Key concepts learned:**
- Normal equation vs. gradient descent: same answer, different scaling tradeoff (O(n³) matrix inversion vs. iterative steps).
- Batch / SGD / mini-batch is a noise-vs-cost tradeoff, not a correctness difference.
- Logistic regression's gradient has the *same form* as linear regression's, with ŷ replaced by σ(Xθ) — a natural future "from scratch" exercise.
- L1 drives weights to exactly zero (sparsity / feature selection); L2 only shrinks them smoothly.
- Two-pass vs. one-pass is a style / constant-factor distinction, not a complexity-class one — both were O(n).

**Result / metric:** GD solution matched the normal equation and sklearn to 4 decimals (final cost 0.2988 after 200 iterations). Lasso correctly zeroed 6 of 10 known-zero weights. Iris: precision/recall/F1 ≈ 0.95–1.00 per class, macro ROC-AUC = 1.0.
