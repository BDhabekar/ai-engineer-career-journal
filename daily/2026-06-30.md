# 2026-06-30 — Day 5

**Phase:** 1 — ML Foundations · **Tracker Day:** 5

## Objective
Turn the by-hand preprocessing from Projects 1–3 into a reusable, leak-free `sklearn` `Pipeline` + `ColumnTransformer`, and prove it reproduces the Day-3 Titanic baseline before adding anything new.

## What I actually did
- New notebook in `02-tabular-ml-project`: `notebooks/02_pipelines_columntransformer_titanic.ipynb` (Day-3 notebook left untouched).
- Re-confirmed the `alive` ≡ `survived` leak via a cross-tab (perfect diagonal 549/342), dropped the derived/leaked columns.
- Built the harness:
  - numeric path → `SimpleImputer(strategy="median", add_indicator=True)` → `StandardScaler()`
  - categorical path → `SimpleImputer(strategy="most_frequent")` → `OneHotEncoder(handle_unknown="ignore")`
  - `ColumnTransformer` routing each, wrapped as `Pipeline([("pre", pre), ("model", m)])`.
- Verified in-session that the pipeline reproduced the Day-3 four-model CV baseline (~0.81–0.82) **before** adding features — the refactor changed plumbing, not results.
- Added the genuinely-new pieces: a missing-`age` indicator, and engineered `family_size = sibsp + parch + 1` and `is_alone`.
- Scored four models on the same 5-fold `StratifiedKFold` by passing the **whole pipeline** to `cross_val_score`.

## Key concepts
- **Fit/apply firewall.** A step belongs *inside* the pipeline iff it **learns a parameter from the data** (median, μ/σ, encoder vocabulary). Row-wise-deterministic features (`family_size`) are leak-safe to compute up front. Nulls are *not* the test — learned-vs-not is.
- `cross_val_score(pipeline, …)` refits every transformer per fold on that fold's training rows only — the real reason a pipeline beats preprocessing the frame once.
- Trees are scale-invariant → the `StandardScaler` was a visible no-op for them ("which models need scaling", seen directly).
- Missingness can be signal → `add_indicator` (the fault-flag move).

## Result
- Transform width = **13** = (7 numeric + 1 age-missing indicator) + (sex 2 + embarked 3 one-hot).
- Feature-included CV: tree(d4) 0.809 ± 0.010 · forest 0.798 ± 0.019 · gboost 0.822 ± 0.035 · xgb 0.809 ± 0.024.
- Held-out test: 0.782 / 0.793 / 0.804 / 0.793 (179 rows, ±~0.03 noise — CV is the trusted comparison).
- **Honest negative result:** the engineered features + indicator did **not** materially move CV — the ensembles already extract that signal from `sibsp`/`parch`/`fare`. Reportable, not a failure.
