**Learned**
- Overfitting is a *train–test gap*, not a low test score (tree: train 0.98 / test 0.82, gap 0.16).
- A single split is too noisy to rank close models (~±0.03); CV reveals the real depth peak (4).
- Bagging (forest: parallel deep trees → cut variance, more trees never hurt) vs boosting (sequential shallow trees → cut bias, too many trees overfit). Same `n_estimators` knob, opposite behaviour.
- Impurity-based feature importance is biased toward continuous features → permutation importance is the unbiased cross-check.
- Mild imbalance (62/38): judge per-class, not by accuracy; threshold tuning moved survivor recall (0.725 → 0.812), class weighting didn't.

**Results (same 5-fold CV):** tree(d4) 0.813±0.013 · RF(300) 0.812±0.024 · GB 0.822±0.031 · XGB(def) 0.801±0.024. All cluster ~0.81–0.82 — honest evaluation mattered more than the model choice.
